# PRODUCT.md — Magic Arena（魔法竞技场）

## 目标（Goal）
一款浏览器端**单机回合制战术RPG**，对标《火焰纹章：封印之剑》的体量与完成度。玩家操控一支法师小队，在完整的世界观中体验**长篇战役、角色成长、阵营冲突与命运抉择**。全部逻辑在本地客户端完成，无需服务端。

## ⚠️ 原创声明（Originality & IP Safety）
- 本作**对标 Fire Emblem 的体量和叙事结构**，但**所有设定、角色名称、地名、剧情、阵营背景均为原创**。
- **禁止直接引用、重写或变体使用任何已有作品的专有名词**（包括但不限于：Fire Emblem/Binding Blade/封印之剑/烈火之剑/圣魔之光石等系列中的角色名、地名、物品名）。
- 世界设定中的"元素魔法+裂隙"框架仅作玩法基础，**剧情走向、角色关系和核心冲突必须完全原创**。
- 所有美术资源（如后续引入）需为原创或使用 CC0 许可证资源。
- 任何 Agent 发现有侵权风险的内容（名称/剧情雷同），必须标记 `BLOCKED` 并提交 `Discussions` 裁定修改。

## 世界设定（World Setting — 完全原创）

**大陆名称**：维尔德兰（Virdlan）大陆

**核心冲突**：千年前，"灵能之潮"（Aether Surge）将大陆从蛮荒时代推入魔法文明。五大灵脉学派——炽焰、凛霜、翠息、冥暗、玄雷——各自发展出独特的魔法体系与社会结构。然而一场"灵脉枯萎"（Aether Decay）正在缓慢侵蚀大地，魔法正在消逝。各大灵脉学派互相指责是对方导致了枯萎，大陆陷入"学派战争"（Schism War）。

**玩家角色**：玩家扮演一名来自边陲之地的**无派系灵能者（Unbound）**，天生具备亲和多派系灵脉的罕见体质。在灵脉枯萎加剧的危机中，被卷入学派战争的漩涡，必须在五大灵脉学派之间斡旋、战斗、选择——修复灵脉，还是接受枯萎？

**主要阵营（五大灵脉学派 + 隐藏势力）**：

| 阵营 | 灵脉 | 社会特征 | 玩法特色 |
|---|---|---|---|
| 炽焰庭（Pyral Keep） | 炽焰 | 军事化城邦，尚武、荣誉至上 | 高攻击、低防御，爆发型单位 |
| 凛霜堡（Cryo Spire） | 凛霜 | 封闭的学术堡垒，理性、冷漠 | 控制型技能、减速/冰冻战术 |
| 翠息林（Verdant Weave） | 翠息 | 自然守护者，隐居森林 | 治疗、增益、地形优势 |
| 冥暗渊（Shade Depth） | 冥暗 | 地下城邦，隐秘、实用主义 | 减益、暗杀、召唤 |
| 玄雷塔（Volt Citadel） | 玄雷 | 工匠与发明家的城市 | 高机动、范围攻击、科技型技能 |
| **蚀教（Eclipse Cult）（隐藏）** | 禁忌灵脉 | 企图利用枯萎的力量重塑世界 | 终章敌人或可选路线 |

**主线剧情纲要**（原创 · 禁止照搬 FE）：

```
序章（1-2 章）：灵能者试炼 → 目睹灵脉枯萎的预兆 → 被迫离开边陲之地
第一幕（3-6 章）：初入学派战争 → 结识各派系角色 → 选择首次阵营立场
第二幕（7-10 章）：灵脉枯萎加速 → 发现蚀教的影子 → 阵营关系开始变化
第三幕（11-14 章）：全面战争爆发 → 关键角色命运转折 → 面对蚀教的真相
终章（15-16 章）：灵脉之源决战 → 根据玩家一路的选择，导向多结局
外传章节：特定角色的个人故事章节，可招募隐藏角色
```

## 核心玩法循环

```
世界地图（选择章节/营地/商店/角色管理）
  │
  ├─ 角色管理（Unit Management）：
  │   招募/编队/装备/技能配置/查看背景故事
  │
  ├─ 营地对话（Camp Dialogue）：
  │   角色间互动事件、支援对话、解锁背景信息
  │
  ├─ 商店（Shop）：
  │   购买装备/消耗品/技能书
  │
  └─ 章节战斗（Battle）：
       部署阵容 → 战前对话/剧情事件 → 回合对战
       → 胜负结算 → 战后剧情事件 → 角色成长
       → 返回世界地图
```

## 角色系统

- **可招募角色 ≥12 名**，每个角色有：名字、称号、灵脉归属、性格描述、背景故事（≥200字）、初始数据、技能列表。
- **角色招募**：通过主线剧情加入、外传章节解锁、特定条件下隐藏招募。
- **角色互动**：章节间随机触发角色间对话事件（类似 FE 支援对话），解锁角色背景故事或获得能力加成。
- **角色成长**：战后获得经验值，升级提升属性，特定等级解锁新技能或转职。
- **角色永久退场（可选）**：战斗中 HP 归零的角色在本场战斗后"重伤离队"，不可再次出战（非死亡，避免严肃题材问题，但产生后果）。

## 章节结构

| 章节 | 类型 | 内容 |
|---|---|---|
| 序章 1-2 | 线性教程 | 基础操作、首位队友加入、世界观导入 |
| 第 1-2 章 | 引入 | 抵达第一座城市、选择初始阵营立场 |
| 第 3-4 章 | 阵营线 | 与所选阵营共同行动，结识该阵营角色 |
| 第 5-6 章 | 转折 | 发现灵脉枯萎的真相线索 |
| 第 7-8 章 | 危机 | 一方阵营遭到重创，被迫逃亡 |
| 第 9-10 章 | 联合 | 尝试团结阵营、招募关键角色 |
| 第 11-12 章 | 决战前 | 蚀教现身、灵脉之源入口开启 |
| 第 13-14 章 | 终局 | 灵脉之源内的连续战斗、多分支结局 |
| 外传 | 可选 | 特定角色个人故事，≥3 条 |

## 约束
- 纯前端、零服务端依赖；不可引入 Express / Socket.IO / 远程 DB / Docker 服务。
- **所有设定必须原创**，禁止直接或变体使用任何已有IP的专有名词。
- 世界观以**文本驱动**（JSON/JS 数据表 + HTML 界面显示），不需要 3D 模型或动画过场。
- 所有剧情/角色/世界内容在 `magic-arena/data/lore/` 目录下以数据表定义，核心逻辑只读数据。
- 提交必须过测试。

## 验收标准
- 打开游戏后显示**世界地图**，有此章节可选，而不是直接进对战。
- 战役路线 ≥12 章，每章有**开场剧情文本**和**结束剧情文本**。
- ≥12 名可招募角色，每人有**完整的背景故事（≥200字）**和**至少 1 个互动对话事件**。
- 存在**角色成长系统**（经验值/等级/属性提升）。
- 存在**战前部署界面**（编组/装备/技能选择）。
- 存在**多结局条件**，根据玩家选择导向至少 2 种不同结局。
- 所有名称/设定经核查**无侵权风险**（与原作无撞名/撞设定）。
